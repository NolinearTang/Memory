# FastAPI 服务中如何处理用户中断请求

## 📋 问题描述

**典型场景**：
- 一个 FastAPI 接口内部有**多个环节**，每个环节可能调用多次大模型
- 处理链路长，总耗时可能达到数十秒甚至数分钟
- 用户在前端等待时可能点击"停止"按钮

**核心挑战**：
1. ⚠️ 环节多且串行，如何在每个环节检查取消状态？
2. ⚠️ 每个环节内部有多次 LLM 调用，如何快速中断？
3. ⚠️ 取消后如何清理已分配的资源（如 token、连接）？
4. ⚠️ 如何保存已完成环节的中间结果？
5. ⚠️ 正在进行的 LLM API 调用如何中止？

**需求**：用户在等待过程中点击停止按钮，需要**尽快**暂停/取消 FastAPI 中正在进行的处理环节。

---

## 🎯 核心解决方案

### 方案概览

FastAPI 基于 Starlette，支持**请求取消检测**。主要方法：

1. **使用 `Request.is_disconnected()`** - 检测客户端是否断开
2. **使用 `asyncio.CancelledError`** - 捕获取消异常
3. **使用后台任务 + 取消令牌** - 主动管理任务生命周期
4. **使用 SSE/WebSocket** - 实时通信和取消控制

---

## 💡 方案一：多环节流水线 + 检查点机制（核心方案）

### 问题场景示例

假设一个智能文档处理服务，有以下环节：

```
用户请求 → 环节1: 提取关键词(3次LLM) → 环节2: 生成摘要(5次LLM) 
        → 环节3: 分析情感(2次LLM) → 环节4: 生成报告(4次LLM) → 返回结果
```

**特点**：
- 总共 4 个环节
- 总共需要 14 次 LLM 调用
- 预计耗时 30-60 秒
- 用户可能在任何时刻点击停止

### 核心设计原则

#### 1. **检查点机制** - 在关键位置检查取消状态

```
每个环节开始前 ✓
每个环节内部的 LLM 调用前后 ✓
每个 LLM 调用内部（如果流式） ✓
```

#### 2. **快速响应** - 检查频率要高

```python
# ❌ 错误：只在环节间检查
环节1 (15秒) → 检查 → 环节2 (15秒) → 检查 → ...
# 最坏情况：用户点停止后还要等 15 秒

# ✅ 正确：环节内部也要检查
环节1: LLM1 → 检查 → LLM2 → 检查 → LLM3 → 检查
# 最坏情况：只需等待单次 LLM 调用（2-5秒）
```

### 完整实现示例

```python
from fastapi import FastAPI, Request, HTTPException
import asyncio
from typing import Optional, Dict, Any
import logging

logger = logging.getLogger(__name__)
app = FastAPI()


class CancelledException(Exception):
    """请求被取消异常"""
    pass


async def check_cancellation(request: Request, stage: str = ""):
    """
    检查请求是否被取消
    
    Args:
        request: FastAPI Request 对象
        stage: 当前所处阶段（用于日志）
    
    Raises:
        CancelledException: 如果请求已被取消
    """
    if await request.is_disconnected():
        logger.info(f"请求在 [{stage}] 阶段被取消")
        raise CancelledException(f"请求在 {stage} 阶段被用户取消")


async def call_llm_api(prompt: str, model: str = "gpt-4") -> str:
    """
    单次 LLM API 调用
    
    注意：这里使用 asyncio.wait_for 设置超时，
    如果任务被取消，会抛出 CancelledError
    """
    try:
        # 实际调用 LLM API（这里用 sleep 模拟）
        # 实际应该是：
        # response = await openai_client.chat.completions.create(...)
        await asyncio.sleep(2)  # 模拟 LLM API 耗时
        return f"LLM response for: {prompt}"
    
    except asyncio.CancelledError:
        logger.info(f"LLM 调用被取消: {prompt[:50]}...")
        raise  # 重新抛出以便上层处理


async def stage_1_extract_keywords(
    text: str, 
    request: Request
) -> Dict[str, Any]:
    """
    环节1: 提取关键词（需要3次LLM调用）
    """
    logger.info("开始环节1: 提取关键词")
    
    # ✅ 环节开始前检查
    await check_cancellation(request, "环节1开始")
    
    keywords = []
    
    # 第1次LLM调用：提取主题
    await check_cancellation(request, "环节1-LLM1前")
    topics = await call_llm_api(f"提取主题: {text[:100]}")
    keywords.append(topics)
    
    # 第2次LLM调用：提取实体
    await check_cancellation(request, "环节1-LLM2前")
    entities = await call_llm_api(f"提取实体: {text[:100]}")
    keywords.append(entities)
    
    # 第3次LLM调用：提取关键短语
    await check_cancellation(request, "环节1-LLM3前")
    phrases = await call_llm_api(f"提取短语: {text[:100]}")
    keywords.append(phrases)
    
    result = {
        "stage": "keywords",
        "keywords": keywords,
        "topics": topics,
        "entities": entities,
        "phrases": phrases
    }
    
    logger.info("环节1完成")
    return result


async def stage_2_generate_summary(
    text: str,
    keywords: Dict[str, Any],
    request: Request
) -> Dict[str, Any]:
    """
    环节2: 生成摘要（需要5次LLM调用）
    """
    logger.info("开始环节2: 生成摘要")
    
    await check_cancellation(request, "环节2开始")
    
    summaries = []
    
    # 5次LLM调用：分段摘要
    for i in range(5):
        await check_cancellation(request, f"环节2-LLM{i+1}前")
        
        summary = await call_llm_api(
            f"摘要第{i+1}部分，关键词: {keywords['topics']}"
        )
        summaries.append(summary)
        
        # ✅ 每次调用后也检查（可选，提高响应速度）
        await check_cancellation(request, f"环节2-LLM{i+1}后")
    
    result = {
        "stage": "summary",
        "summaries": summaries,
        "final_summary": " ".join(summaries)
    }
    
    logger.info("环节2完成")
    return result


async def stage_3_analyze_sentiment(
    text: str,
    summary: Dict[str, Any],
    request: Request
) -> Dict[str, Any]:
    """
    环节3: 情感分析（需要2次LLM调用）
    """
    logger.info("开始环节3: 情感分析")
    
    await check_cancellation(request, "环节3开始")
    
    # 第1次：总体情感
    await check_cancellation(request, "环节3-LLM1前")
    overall = await call_llm_api(f"分析情感: {summary['final_summary'][:100]}")
    
    # 第2次：细粒度情感
    await check_cancellation(request, "环节3-LLM2前")
    detailed = await call_llm_api(f"细粒度情感: {text[:100]}")
    
    result = {
        "stage": "sentiment",
        "overall_sentiment": overall,
        "detailed_sentiment": detailed
    }
    
    logger.info("环节3完成")
    return result


async def stage_4_generate_report(
    keywords: Dict[str, Any],
    summary: Dict[str, Any],
    sentiment: Dict[str, Any],
    request: Request
) -> Dict[str, Any]:
    """
    环节4: 生成报告（需要4次LLM调用）
    """
    logger.info("开始环节4: 生成报告")
    
    await check_cancellation(request, "环节4开始")
    
    report_parts = []
    
    # 4次LLM调用：生成报告的不同部分
    sections = ["introduction", "key_points", "analysis", "conclusion"]
    
    for i, section in enumerate(sections):
        await check_cancellation(request, f"环节4-{section}前")
        
        part = await call_llm_api(
            f"生成{section}: 关键词={keywords['topics']}, "
            f"摘要={summary['final_summary'][:50]}"
        )
        report_parts.append({section: part})
    
    result = {
        "stage": "report",
        "sections": report_parts,
        "full_report": "\n\n".join([str(p) for p in report_parts])
    }
    
    logger.info("环节4完成")
    return result


@app.post("/api/process-document")
async def process_document(
    request: Request,
    text: str
):
    """
    处理文档的完整流程（4个环节，14次LLM调用）
    """
    # 用于存储中间结果
    intermediate_results = {
        "completed_stages": [],
        "partial_results": {}
    }
    
    try:
        # 环节1: 提取关键词 (3次LLM)
        keywords = await stage_1_extract_keywords(text, request)
        intermediate_results["completed_stages"].append("keywords")
        intermediate_results["partial_results"]["keywords"] = keywords
        
        # 环节2: 生成摘要 (5次LLM)
        summary = await stage_2_generate_summary(text, keywords, request)
        intermediate_results["completed_stages"].append("summary")
        intermediate_results["partial_results"]["summary"] = summary
        
        # 环节3: 情感分析 (2次LLM)
        sentiment = await stage_3_analyze_sentiment(text, summary, request)
        intermediate_results["completed_stages"].append("sentiment")
        intermediate_results["partial_results"]["sentiment"] = sentiment
        
        # 环节4: 生成报告 (4次LLM)
        report = await stage_4_generate_report(
            keywords, summary, sentiment, request
        )
        intermediate_results["completed_stages"].append("report")
        intermediate_results["partial_results"]["report"] = report
        
        # 全部完成
        return {
            "status": "success",
            "completed_stages": intermediate_results["completed_stages"],
            "results": intermediate_results["partial_results"],
            "final_report": report["full_report"]
        }
    
    except CancelledException as e:
        # 用户取消了请求
        logger.info(f"请求被取消: {str(e)}")
        return {
            "status": "cancelled",
            "message": str(e),
            "completed_stages": intermediate_results["completed_stages"],
            "partial_results": intermediate_results["partial_results"],
            "note": "已保存部分完成的结果"
        }
    
    except asyncio.CancelledError:
        # asyncio 层面的取消
        logger.info("任务在 asyncio 层被取消")
        return {
            "status": "cancelled",
            "completed_stages": intermediate_results["completed_stages"],
            "partial_results": intermediate_results["partial_results"]
        }
    
    except Exception as e:
        # 其他错误
        logger.error(f"处理失败: {str(e)}", exc_info=True)
        return {
            "status": "error",
            "error": str(e),
            "completed_stages": intermediate_results["completed_stages"],
            "partial_results": intermediate_results["partial_results"]
        }
```

### 关键设计要点

#### 1. **检查点密度**

```python
# 建议的检查点位置：
✓ 每个环节开始前
✓ 每次 LLM 调用前
✓ （可选）每次 LLM 调用后
✓ （可选）长循环内部

# 响应时间计算：
最大响应延迟 = 单次 LLM 调用时间
例如：单次 LLM 2-5 秒 → 用户点停止后 2-5 秒内响应
```

#### 2. **中间结果保存**

```python
# ✅ 使用字典保存每个环节的结果
intermediate_results = {
    "completed_stages": [],  # 已完成的环节
    "partial_results": {}    # 每个环节的结果
}

# 好处：
# 1. 用户取消后可以看到已完成的部分
# 2. 可以支持"继续"功能
# 3. 便于调试和监控
```

#### 3. **异常处理层次**

```python
try:
    # 业务逻辑
    await stage_1()
    await stage_2()
except CancelledException:
    # 业务层取消（用户主动取消）
    return partial_results
except asyncio.CancelledError:
    # asyncio 层取消（系统级）
    cleanup()
    raise
except Exception:
    # 其他错误
    log_error()
    return error_response
```

### 使用场景

✅ **适用于**：
- ✅ 多环节串行处理（3-10个环节）
- ✅ 每个环节有多次 LLM 调用
- ✅ 需要保存中间结果
- ✅ 总耗时 >10 秒的长流程

❌ **局限性**：
- ❌ 无法中断**正在执行的** LLM API 调用
- ❌ 最快响应时间 = 单次 LLM 调用时间（2-5秒）
- ❌ 不适合需要 < 1秒响应的场景

## 💡 方案二：真正中断 LLM API 调用（使用 httpx 超时和取消）

### 问题：方案一的局限

方案一虽然能在LLM调用**之间**检测取消，但无法中断**正在进行**的LLM API调用：

```python
# 当前状态：
await check_cancellation(request, "LLM1前")  # ✅ 可以检测
await call_llm_api(prompt)  # ❌ 这里需要2-5秒，期间无法中断
await check_cancellation(request, "LLM1后")  # ✅ 可以检测
```

**问题**：用户点击停止后，最坏情况需要等待单次LLM调用完成（2-5秒）。

### 解决方案：使用支持取消的HTTP客户端

使用 `httpx` 替代 `requests`，它支持：
1. 异步HTTP请求
2. 超时控制
3. 请求取消

### 完整实现

```python
import httpx
import asyncio
from fastapi import FastAPI, Request
from typing import Dict, Any
import logging

logger = logging.getLogger(__name__)
app = FastAPI()


class LLMClient:
    """支持取消的 LLM 客户端"""
    
    def __init__(self, api_key: str, base_url: str = "https://api.openai.com/v1"):
        # 使用 httpx.AsyncClient，支持异步和取消
        self.client = httpx.AsyncClient(
            timeout=30.0,  # 默认30秒超时
            limits=httpx.Limits(
                max_keepalive_connections=5,
                max_connections=10
            )
        )
        self.api_key = api_key
        self.base_url = base_url
    
    async def call_llm(
        self,
        prompt: str,
        model: str = "gpt-4",
        task: asyncio.Task = None  # 传入任务引用，用于检查取消
    ) -> str:
        """
        调用 LLM API，支持取消
        
        Args:
            prompt: 提示词
            model: 模型名称
            task: asyncio.Task 引用（可选，用于检查是否被取消）
        
        Returns:
            LLM 响应
        
        Raises:
            asyncio.CancelledError: 任务被取消
            httpx.TimeoutException: 超时
        """
        headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json"
        }
        
        payload = {
            "model": model,
            "messages": [{"role": "user", "content": prompt}],
            "temperature": 0.7
        }
        
        try:
            # 使用 asyncio.create_task 包装请求，以便可以取消
            response_task = asyncio.create_task(
                self.client.post(
                    f"{self.base_url}/chat/completions",
                    json=payload,
                    headers=headers,
                    timeout=30.0  # 单次请求30秒超时
                )
            )
            
            # 等待响应，同时检查是否被取消
            while not response_task.done():
                # 检查父任务是否被取消
                if task and task.cancelled():
                    response_task.cancel()  # 取消 HTTP 请求
                    raise asyncio.CancelledError("LLM 调用被取消")
                
                # 短暂等待，避免占用过多CPU
                await asyncio.sleep(0.1)
            
            response = await response_task
            response.raise_for_status()
            
            data = response.json()
            return data["choices"][0]["message"]["content"]
        
        except asyncio.CancelledError:
            logger.info(f"LLM 调用被取消: {prompt[:50]}...")
            raise
        
        except httpx.TimeoutException:
            logger.error(f"LLM 调用超时: {prompt[:50]}...")
            raise
        
        except Exception as e:
            logger.error(f"LLM 调用失败: {str(e)}")
            raise
    
    async def close(self):
        """关闭客户端"""
        await self.client.aclose()


# 全局 LLM 客户端
llm_client = None


@app.on_event("startup")
async def startup():
    global llm_client
    llm_client = LLMClient(api_key="your-api-key")


@app.on_event("shutdown")
async def shutdown():
    global llm_client
    if llm_client:
        await llm_client.close()


async def process_with_cancellable_llm(
    text: str,
    request: Request
) -> Dict[str, Any]:
    """
    处理流程，支持取消（包括取消正在进行的 LLM 调用）
    """
    # 获取当前任务引用
    current_task = asyncio.current_task()
    
    results = []
    
    try:
        # 创建后台任务来监听客户端断开
        async def monitor_disconnect():
            """监听客户端断开，如果断开则取消主任务"""
            while True:
                if await request.is_disconnected():
                    logger.info("检测到客户端断开，取消任务")
                    current_task.cancel()
                    break
                await asyncio.sleep(0.5)  # 每0.5秒检查一次
        
        monitor_task = asyncio.create_task(monitor_disconnect())
        
        try:
            # 环节1：多次LLM调用
            for i in range(5):
                logger.info(f"LLM 调用 {i+1}/5")
                
                # 调用 LLM，传入任务引用以便在内部检查取消
                result = await llm_client.call_llm(
                    f"处理第{i+1}部分: {text[:100]}",
                    task=current_task
                )
                
                results.append(result)
                
                # 如果任务被取消，这里会抛出 CancelledError
                await asyncio.sleep(0)  # 让出控制权，检查取消状态
        
        finally:
            # 停止监听任务
            monitor_task.cancel()
            try:
                await monitor_task
            except asyncio.CancelledError:
                pass
        
        return {
            "status": "success",
            "results": results
        }
    
    except asyncio.CancelledError:
        logger.info("处理被取消")
        return {
            "status": "cancelled",
            "completed": len(results),
            "partial_results": results
        }


@app.post("/api/process-cancellable")
async def process_cancellable(
    request: Request,
    text: str
):
    """支持真正取消的处理接口"""
    return await process_with_cancellable_llm(text, request)
```

### 关键改进点

#### 1. **HTTP 请求级别的取消**

```python
# ✅ 包装 HTTP 请求为 Task
response_task = asyncio.create_task(
    client.post(...)
)

# ✅ 在循环中检查是否被取消
while not response_task.done():
    if parent_task.cancelled():
        response_task.cancel()  # 立即取消 HTTP 请求
        raise CancelledError
    await asyncio.sleep(0.1)

# 好处：
# - 可以在 LLM API 调用过程中取消
# - 响应时间 < 1 秒（检查间隔0.1秒）
# - 不会浪费 API 调用费用
```

#### 2. **后台监听客户端断开**

```python
async def monitor_disconnect():
    """持续监听客户端是否断开"""
    while True:
        if await request.is_disconnected():
            main_task.cancel()  # 取消主任务
            break
        await asyncio.sleep(0.5)

# 优点：
# - 不需要在每个位置手动检查
# - 自动传播取消信号
# - 代码更简洁
```

#### 3. **使用超时保护**

```python
# 为每个 LLM 调用设置超时
await client.post(..., timeout=30.0)

# 为整个流程设置超时
result = await asyncio.wait_for(
    process_pipeline(...),
    timeout=120.0  # 整个流程最多120秒
)
```

### 使用场景

✅ **优势**：
- ✅ 可以**真正中断**正在进行的 LLM API 调用
- ✅ 响应时间 < 1 秒（检查间隔可调）
- ✅ 节省 API 调用成本
- ✅ 用户体验更好

⚠️ **注意事项**：
- ⚠️ 需要使用 `httpx` 而不是 `requests`
- ⚠️ 需要正确处理取消异常
- ⚠️ 需要在 `finally` 中清理资源

## 💡 方案三：使用 asyncio.Task + 取消令牌（推荐用于复杂场景）

### 原理

将大模型调用包装成 `asyncio.Task`，当检测到客户端断开时，主动取消任务。

### 实现代码

```python
from fastapi import FastAPI, Request, BackgroundTasks
import asyncio
from typing import Dict
import uuid

app = FastAPI()

# 全局任务管理器
active_tasks: Dict[str, asyncio.Task] = {}


async def llm_pipeline(task_id: str, prompts: list[str], request: Request):
    """大模型处理流水线"""
    results = []
    
    try:
        for i, prompt in enumerate(prompts):
            # 定期检查是否被取消
            if await request.is_disconnected():
                print(f"任务 {task_id} 被用户取消")
                break
            
            print(f"任务 {task_id} - 处理 {i+1}/{len(prompts)}")
            
            # 模拟 LLM 调用（可中断）
            try:
```

## 💡 方案三：使用流式响应（SSE）+ 取消控制

### 原理

使用服务器发送事件（SSE）实现流式响应，客户端可随时断开连接，服务器实时检测。

### 实现代码

```python
from fastapi import FastAPI, Request
from fastapi.responses import StreamingResponse
import asyncio
import json

app = FastAPI()


async def llm_stream_generator(prompts: list[str], request: Request):
    """流式生成 LLM 结果"""
    
    for i, prompt in enumerate(prompts):
        # 检查客户端是否断开
        if await request.is_disconnected():
            yield f"data: {json.dumps({'status': 'cancelled', 'step': i})}\n\n"
            break
        
        # 发送进度
        yield f"data: {json.dumps({'status': 'processing', 'step': i, 'total': len(prompts)})}\n\n"
        
        # 调用 LLM
        try:
            result = await call_single_llm(prompt)
            
            # 发送结果
            yield f"data: {json.dumps({'status': 'step_complete', 'step': i, 'result': result})}\n\n"
            
        except asyncio.CancelledError:
            yield f"data: {json.dumps({'status': 'cancelled', 'step': i})}\n\n"
            break
        except Exception as e:
            yield f"data: {json.dumps({'status': 'error', 'step': i, 'error': str(e)})}\n\n"
            break
    
    # 发送完成信号
    yield f"data: {json.dumps({'status': 'done'})}\n\n"


async def call_single_llm(prompt: str) -> str:
    """单次 LLM 调用"""
    await asyncio.sleep(2)
    return f"回复: {prompt}"


@app.get("/stream-process")
async def stream_process(request: Request, count: int = 5):
    """流式处理请求"""
    prompts = [f"问题 {i+1}" for i in range(count)]
    
    return StreamingResponse(
        llm_stream_generator(prompts, request),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "Connection": "keep-alive",
        }
    )
```

### 实现代码

```python
from fastapi import FastAPI, WebSocket, WebSocketDisconnect
import asyncio
import json

app = FastAPI()


@app.websocket("/ws/process")
async def websocket_process(websocket: WebSocket):
    """WebSocket 处理端点"""
    await websocket.accept()
    
    task = None
    cancelled = False
    
    try:
        # 接收初始请求
        data = await websocket.receive_json()
        prompts = data.get("prompts", [])
        
        # 创建处理任务
        async def process_with_feedback():
            nonlocal cancelled
            
            for i, prompt in enumerate(prompts):
                if cancelled:
                    await websocket.send_json({
                        "status": "cancelled",
                        "step": i
                    })
                    break
                
                # 发送进度
                await websocket.send_json({
                    "status": "processing",
                    "step": i,
                    "total": len(prompts)
                })
                
                # 调用 LLM
                result = await call_single_llm(prompt)
                
                # 发送结果
                await websocket.send_json({
                    "status": "step_complete",
                    "step": i,
                    "result": result
                })
            
            if not cancelled:
                await websocket.send_json({"status": "done"})
        
        # 启动处理任务
        task = asyncio.create_task(process_with_feedback())
        
        # 监听客户端消息（如取消指令）
        while True:
            try:
                message = await asyncio.wait_for(
                    websocket.receive_json(),
                    timeout=0.1
                )
                
                if message.get("action") == "cancel":
                    cancelled = True
                    if task and not task.done():
                        task.cancel()
                    break
                    
            except asyncio.TimeoutError:
                # 没有新消息，继续检查任务状态
                if task and task.done():
                    break
                continue
        
        # 等待任务完成
        if task:
            try:
                await task
            except asyncio.CancelledError:
                pass
    
    except WebSocketDisconnect:
        print("客户端断开连接")
        if task and not task.done():
            task.cancel()
    
    finally:
        if task and not task.done():
            task.cancel()


async def call_single_llm(prompt: str) -> str:
    """单次 LLM 调用"""
    await asyncio.sleep(2)
    return f"回复: {prompt}"
```

```python
class ResourceManager:
    """资源管理器"""
    
    def __init__(self):
        self.http_client = None
        self.temp_files = []
        self.active_tasks = []
    
    async def __aenter__(self):
        self.http_client = httpx.AsyncClient()
        return self
    
    async def __aexit__(self, exc_type, exc_val, exc_tb):
        # ✅ 自动清理资源
        
        # 取消所有活动任务
        for task in self.active_tasks:
            if not task.done():
                task.cancel()
        
        # 关闭HTTP客户端
        if self.http_client:
            await self.http_client.aclose()
        
        # 删除临时文件
        for file in self.temp_files:
            try:
                os.remove(file)
            except:
                pass


# 使用
```

## ✅ 核心要点总结

### 问题关键

在多环节、多次LLM调用的长流程中实现用户取消：

1. **检查频率是关键** - 不能只在环节间检查，必须在**每次LLM调用前后**检查
2. **响应速度取决于粒度** - 检查点越密集，响应越快
3. **中间结果要保存** - 取消后返回已完成部分
4. **资源清理要完整** - 使用 `try-finally` 或上下文管理器

### 三种方案速查

| 需求 | 推荐方案 | 响应时间 | 实现难度 |
|------|---------|----------|----------|
| 简单流程（<30秒） | 方案一：检查点 | 2-5秒 | ⭐⭐ |
| 复杂流程（>30秒） | 方案二：httpx取消 | <1秒 | ⭐⭐⭐ |
| 长任务+进度反馈 | 方案三：SSE流式 | <1秒 | ⭐⭐⭐ |

### 实现模板（直接可用）

```python
from fastapi import FastAPI, Request
import asyncio
import httpx

app = FastAPI()

class CancelledException(Exception):
    pass

async def check_cancellation(request: Request, stage: str = ""):
    """检查点函数"""
    if await request.is_disconnected():
        raise CancelledException(f"取消于{stage}")

class LLMClient:
    """支持取消的LLM客户端"""
    def __init__(self):
        self.client = httpx.AsyncClient(timeout=30.0)
    
    async def call(self, prompt: str, parent_task=None):
        task = asyncio.create_task(
            self.client.post("LLM_API_URL", json={...})
        )
        # 轮询检查取消状态
        while not task.done():
            if parent_task and parent_task.cancelled():
                task.cancel()
                raise CancelledException()
            await asyncio.sleep(0.1)  # 0.1秒检查一次
        return await task


@app.post("/api/process")
async def process(request: Request, data: dict):
    """多环节处理（支持取消）"""
    llm = LLMClient()
    results = []
    current_task = asyncio.current_task()
    
    try:
        # 环节1：提取关键词
        await check_cancellation(request, "环节1前")
        kw = await llm.call("提取关键词", current_task)
        results.append(kw)
        
        # 环节2：生成摘要
        await check_cancellation(request, "环节2前")
        summary = await llm.call("生成摘要", current_task)
        results.append(summary)
        
        # 环节3：分析情感
        await check_cancellation(request, "环节3前")
        sentiment = await llm.call("分析情感", current_task)
        results.append(sentiment)
        
        return {"status": "success", "results": results}
    
    except CancelledException as e:
        return {
            "status": "cancelled",
            "message": str(e),
            "partial_results": results  # 返回已完成部分
        }
    finally:
        await llm.client.aclose()  # 清理资源
```

### 核心要点记忆卡

**✅ 必须做的**：
- 在每个环节开始前检查取消状态
- 在每次LLM调用前后检查（提高响应速度）
- 保存中间结果，取消时返回
- 使用 try-finally 清理资源

**❌ 不要做的**：
- 只在环节间检查（响应太慢）
- 吞掉 CancelledError 异常
- 使用阻塞调用（time.sleep）
- 忘记关闭 HTTP 客户端

**响应时间公式**：
```
最大响应延迟 = min(单次LLM调用时间, 检查间隔)

方案一（只在LLM间检查）：延迟 = 2-5秒
方案二（LLM内部轮询）：延迟 = 0.1秒
```

## 🎓 总结

### 问题本质

在 FastAPI 服务中处理多环节、多次 LLM 调用的长流程，用户点击停止后如何快速响应？

### 核心解决方案

1. **检查频率** - 在每个环节前、每次 LLM 调用前后检查 `request.is_disconnected()`
2. **真正中断** - 使用 `httpx.AsyncClient` 并在循环中检查 `task.cancelled()`，可中断正在进行的 HTTP 请求
3. **保存进度** - 使用字典保存每个环节的结果，取消时返回部分结果
4. **清理资源** - 使用 `try-finally` 或上下文管理器确保资源释放

### 方案选择

| 场景 | 推荐方案 | 响应时间 | 复杂度 |
|------|---------|----------|--------|
| 简单流程（<30秒，3-5环节） | 方案一：检查点机制 | 2-5秒 | ⭐⭐ |
| 复杂流程（>30秒，>5环节） | 方案二：httpx取消 | <1秒 | ⭐⭐⭐ |
| 长任务（>60秒，需进度） | 方案三：SSE流式 | <1秒 | ⭐⭐⭐ |
